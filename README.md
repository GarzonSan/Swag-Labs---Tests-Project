## Objective
I've chosen this application to display the skills I have in developing a complete test suite. Not only focusing in the tests or in the automations only, but showing a complete set of skills that are used during the engineering of a test environment, such as, planning, documenting, and understanding of the necessary pipeline behind this field of work.

## Focus
The following modules of the application will be covered in this test:
* Login Page
* Store Page
* Shopping Cart
* Checkout Page

### Tools
I've used the **Robot Framework** with the **Python** language, and the **Selenium Library** for this project.

### Approach
This platform will be tested to ensure that the entire E2E purchase scenario is working as intended, as well as guarantee that the modules that complement the user experience within the application are working well together.
It will be using automated black-box tests to simulate the user's interaction with the platform.

## Good Practices
I'm still using the **Page Object Oriented Pattern** to keep the project in the most organized and easy to maintain way, since this time I'm working with an demo application that have more integrated systems, and keeping things separated this way helps when adapting the tests for new features or design changes in the application.
